# Изменение: Тег "Все" вместо кнопки "Очистить фильтры"

## Что изменилось

Заменена кнопка "Очистить фильтры" на тег "Все" в списке тегов.

## До

```
Теги:
┌─────────┐ ┌──────────┐ ┌────────┐
│ дизайн  │ │фронтенд  │ │ бэкенд │
└─────────┘ └──────────┘ └────────┘

┌──────────────────┐
│✕ Очистить фильтры│
└──────────────────┘
```

## После

```
Теги:
┌─────┐ ┌─────────┐ ┌──────────┐ ┌────────┐
│ Все │ │ дизайн  │ │фронтенд  │ │ бэкенд │
└─────┘ └─────────┘ └──────────┘ └────────┘
```

## Преимущества

✅ **Интуитивнее** - тег "Все" более понятен пользователям  
✅ **Компактнее** - не нужна отдельная секция для очистки  
✅ **Единообразие** - все фильтры в одном стиле  
✅ **Меньше кликов** - тег "Все" всегда на виду  
✅ **Лучший UX** - явное состояние "показать все"

## Поведение

### Тег "Все"

- **Активен по умолчанию** - когда не выбран ни один тег
- **Клик на "Все"** - сбрасывает выбранный тег
- **Визуально** - такой же стиль, как и другие теги

### Другие теги

- **Клик на тег** - выбирает этот тег и снимает "Все"
- **Повторный клик** - НЕ снимает тег (в отличие от старого поведения)
- **Только один тег** - можно выбрать только один тег за раз

## Изменения в коде

### 1. Обновлен метод selectTag

**Файл:** `src/app/components/course-filters/course-filters.ts`

**До:**

```typescript
selectTag(tag: string): void {
  const newTag = this.selectedTag === tag ? null : tag;
  this.selectedTag = newTag;
  this.emitFilters();
}
```

**После:**

```typescript
selectTag(tag: string | null): void {
  this.selectedTag = tag;
  this.emitFilters();
}
```

**Изменения:**

- Принимает `string | null` вместо `string`
- Убрана логика toggle (повторный клик не снимает тег)
- Просто устанавливает выбранный тег

### 2. Удален метод clearFilters

**Файл:** `src/app/components/course-filters/course-filters.ts`

**Удалено:**

```typescript
clearFilters(): void {
  this.selectedTag = null;
  this.selectedSort = null;
  this.emitFilters();
}
```

Теперь очистка тега происходит через `selectTag(null)`.

## Логика работы

### Сценарий 1: Пользователь заходит на страницу

```
1. selectedTag = null
2. Тег "Все" активен (мятный фон)
3. Показываются все курсы
```

### Сценарий 2: Пользователь выбирает тег "фронтенд"

```
1. Клик на "фронтенд"
2. selectedTag = "фронтенд"
3. Тег "Все" становится неактивным
4. Тег "фронтенд" становится активным
5. Показываются только курсы с тегом "фронтенд"
```

### Сценарий 3: Пользователь хочет вернуться ко всем курсам

```
1. Клик на "Все"
2. selectedTag = null
3. Тег "фронтенд" становится неактивным
4. Тег "Все" становится активным
5. Показываются все курсы
```

### Сценарий 4: Пользователь переключается между тегами

```
1. Выбран "фронтенд"
2. Клик на "бэкенд"
3. selectedTag = "бэкенд"
4. Тег "фронтенд" становится неактивным
5. Тег "бэкенд" становится активным
6. Показываются только курсы с тегом "бэкенд"
```

## Сравнение подходов

### Старый подход (кнопка "Очистить")

```
Плюсы:
- Явная кнопка для очистки
- Можно очистить все фильтры сразу

Минусы:
- Занимает дополнительное место
- Появляется/исчезает (может сбивать с толку)
- Отдельный стиль кнопки
- Дополнительная секция в UI
```

### Новый подход (тег "Все")

```
Плюсы:
- Компактнее
- Единообразный стиль
- Всегда на виду
- Интуитивнее для пользователей
- Явное состояние "показать все"

Минусы:
- Не очищает сортировку (только тег)
```

## Примечание о сортировке

Тег "Все" **не сбрасывает сортировку**. Это сделано намеренно, потому что:

1. Сортировка и фильтрация - независимые операции
2. Пользователь может хотеть видеть все курсы, но отсортированные
3. Для сброса сортировки нужно кликнуть на активную сортировку повторно

Если нужно сбросить и тег, и сортировку, можно:

1. Кликнуть на "Все"
2. Кликнуть на активную сортировку
