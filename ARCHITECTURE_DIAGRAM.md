# Архитектура унифицированного компонента курсов

## Схема маршрутизации

```
┌─────────────────────────────────────────────────────────────┐
│                      URL МАРШРУТЫ                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
        ┌─────────────────────────────────────────┐
        │         /courses (основной)              │
        │  Query params: ?scope=me&tag=...&q=...  │
        └─────────────────────────────────────────┘
                              │
                              ▼
        ┌─────────────────────────────────────────┐
        │          CoursesPage Component           │
        │                                          │
        │  • Проверяет route.data.redirectToScope │
        │  • Читает query параметры               │
        │  • Определяет роль пользователя         │
        │  • Загружает соответствующие курсы      │
        └─────────────────────────────────────────┘
                              │
                ┌─────────────┴─────────────┐
                ▼                           ▼
    ┌───────────────────┐       ┌───────────────────┐
    │   СТУДЕНТ         │       │   ПРЕПОДАВАТЕЛЬ   │
    └───────────────────┘       └───────────────────┘
                │                           │
        ┌───────┴───────┐           ┌───────┴───────┐
        ▼               ▼           ▼               ▼
    scope=me      scope=all     scope=me      scope=all
        │               │           │               │
        ▼               ▼           ▼               ▼
  Записанные    Все курсы +   Созданные      Все курсы
    курсы      кнопки записи    курсы
                                  +
                            "Create course"
```

## Сценарии использования

### Сценарий 1: Студент просматривает все курсы

```
URL: /courses
↓
CoursesPage
↓
scope = null
role = STUDENT
↓
loadCoursesWithEnrollmentStatus()
↓
Отображение:
• Все курсы
• Кнопки "Записаться"/"Отписаться"
• Нет кнопки "Create course"
```

### Сценарий 2: Студент просматривает свои курсы

```
URL: /courses?scope=me
↓
CoursesPage
↓
scope = 'me'
role = STUDENT
↓
getEnrolledCourses()
↓
Отображение:
• Только записанные курсы
• Нет кнопок записи
• Нет кнопки "Create course"
```

### Сценарий 3: Преподаватель просматривает свои курсы

```
URL: /courses?scope=me
↓
CoursesPage
↓
scope = 'me'
role = TEACHER
↓
getMyCourses()
↓
Отображение:
• Только созданные курсы
• Кнопка "Create course"
• Нет кнопок записи
```

### Сценарий 4: Преподаватель создает курс

```
URL: /courses?scope=me
↓
Клик "Create course"
↓
URL: /courses/create
↓
CreateCoursePage
↓
Заполнение формы
↓
Submit
↓
URL: /courses/:courseId
↓
CourseDetailsPage
```

## Обратная совместимость

```
Старый URL                    Новый URL
─────────────────────────────────────────────────
/my-courses/teaching      →   /courses?scope=me
/my-courses/learning      →   /courses?scope=me
/my-courses/teaching/create → /courses/create

Механизм:
1. Router активирует CoursesPage
2. Constructor проверяет route.data.redirectToScope
3. Если найден → router.navigate с queryParams
4. replaceUrl: true (не добавляет в историю)
```

## Преимущества архитектуры

✅ **Единая точка входа** - один компонент для всех сценариев  
✅ **Реактивность** - автоматическое обновление при изменении параметров  
✅ **Расширяемость** - легко добавлять новые фильтры  
✅ **Переиспользование** - меньше дублирования кода  
✅ **Производительность** - изменение параметров не перезагружает компонент  
✅ **Тестируемость** - один компонент = один набор тестов
