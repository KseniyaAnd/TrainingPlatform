# Админ-панель

## Обзор

Админ-панель предоставляет администраторам платформы полный контроль над пользователями, курсами и статистикой.

## Доступ

Админ-панель доступна только пользователям с ролью `ADMIN`. Доступ защищен `adminGuard`, который автоматически перенаправляет неавторизованных пользователей на главную страницу.

**URL:** `/admin`

## Структура

### 1. Дашборд (`/admin`)

Главная страница админ-панели с общей статистикой платформы.

**Компонент:** `AdminDashboardComponent`

**API эндпоинт:** `GET /platform-statistics`

**Отображаемые данные:**

- Общее количество пользователей
- Общее количество курсов
- Общее количество записей на курсы
- Средний балл по всем заданиям

**Функционал:**

- Карточки со статистикой (кликабельные для навигации)
- Быстрые действия для перехода к управлению пользователями и курсами

### 2. Управление пользователями (`/admin/users`)

Список всех пользователей с возможностью фильтрации по ролям.

**Компонент:** `AdminUsersComponent`

**API эндпоинт:** `GET /users?role={ADMIN|TEACHER|STUDENT}`

**Функционал:**

- Фильтрация по ролям (Все, Администраторы, Преподаватели, Студенты)
- Таблица с информацией о пользователях:
  - Имя пользователя (с аватаром)
  - Email
  - Роль (с цветовой индикацией)
  - Дата регистрации
- Кнопка "Детали" для просмотра подробной информации

### 3. Детали пользователя (`/admin/users/:userId`)

Подробная информация о конкретном пользователе.

**Компонент:** `AdminUserDetailsComponent`

**API эндпоинт:** `GET /users/{userId}/details`

**Отображаемые данные:**

#### Основная информация:

- Имя пользователя
- Email
- Роль
- Дата регистрации

#### Записи на курсы:

- Название курса
- Дата записи
- Прогресс (в процентах с визуальной полосой)

#### Ответы на задания:

- Название задания
- Курс
- Дата отправки
- Оценка (с цветовой индикацией: зеленый ≥80, желтый ≥60, красный <60)

#### Созданные курсы (для преподавателей):

- Название курса
- Дата создания
- Количество записей

### 4. Управление курсами (`/admin/courses`)

Список всех курсов с возможностью редактирования и удаления.

**Компонент:** `AdminCoursesComponent`

**API эндпоинты:**

- `GET /courses` - получение списка курсов
- `DELETE /courses/{courseId}` - удаление курса

**Функционал:**

- Карточки курсов с информацией:
  - Название
  - Описание
  - Теги
  - Количество записей
  - Дата создания
- Действия для каждого курса:
  - **Просмотр** - переход на страницу курса
  - **Редактировать** - переход на страницу курса в режиме редактирования
  - **Удалить** - удаление курса с подтверждением

## Права доступа

Администратор имеет доступ ко всем эндпоинтам API:

### Эксклюзивный эндпоинт:

- `GET /platform-statistics` - статистика платформы

### Пользователи:

- `GET /users` - список всех пользователей с фильтром по роли
- `GET /users/{userId}` - профиль любого пользователя
- `GET /users/{userId}/details` - полный профиль с enrollments, курсами, submissions

### Курсы:

- `GET /courses` - все курсы
- `GET /courses/{courseId}` - любой курс
- `PATCH /courses/{courseId}` - редактировать любой курс
- `DELETE /courses/{courseId}` - удалить любой курс
- `GET /courses/me` - курсы где админ является создателем

### Уроки и лекции:

- `POST /lessons` - создать урок в любом курсе
- `PATCH /lessons/{lessonId}` - редактировать любой урок
- `DELETE /lessons/{lessonId}` - удалить любой урок
- `POST /lectures` - создать лекцию
- `PATCH /lectures/{lectureId}` - редактировать любую лекцию
- `DELETE /lectures/{lectureId}` - удалить любую лекцию

### Ассессменты и submissions:

- `POST /assessments` - создать ассессмент
- `PATCH /submissions/{id}` - выставить оценку за любой submission
- `GET /submissions/assessment/{assessmentId}` - все ответы по ассессменту

### AI аналитика:

- `GET /courses/{courseId}/ai/student-analytics` - аналитика по курсу
- `GET /courses/{courseId}/students/{studentId}/ai-study-plan` - план студента

## Навигация

Кнопка "Админ панель" отображается в меню пользователя (user-menu) только для пользователей с ролью `ADMIN`.

## Безопасность

- Все маршруты админ-панели защищены `adminGuard`
- Guard проверяет роль пользователя через `AuthStateService`
- При попытке доступа без прав администратора происходит редирект на главную страницу
- Все API запросы автоматически включают JWT токен через `authInterceptor`

## Стилизация

Админ-панель использует единый стиль с остальным приложением:

- Цветовая схема: синий (#3b82f6) для основных действий
- Адаптивный дизайн с grid-layout
- Анимации и transitions для улучшения UX
- Иконки из библиотеки Heroicons (SVG)

## Использование

1. Войдите в систему как пользователь с ролью `ADMIN`
2. В меню пользователя появится кнопка "Админ панель"
3. Нажмите на кнопку для перехода на дашборд
4. Используйте навигацию для управления пользователями и курсами
