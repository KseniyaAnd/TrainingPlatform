# Реализация отделов лекций (Lecture Sections)

## Обзор

Добавлена поддержка отделов лекций - структурных единиц, объединяющих несколько лекций внутри курса.

## Иерархия

```
Курс (Course)
├── Урок (Lesson, kind='lesson')
│   └── Лекции (Lectures)
└── Отдел лекций (Lesson, kind='lecture')
    └── Лекции (Lectures)
```

## Модель данных

Отделы лекций используют ту же модель `Lesson`, но с `kind: 'lecture'`:

```typescript
interface Lesson {
  id: string;
  courseId: string;
  title: string;
  kind: 'lecture' | 'lesson';
  content?: string | null;
}
```

## Функциональность

### Уровень отдела лекций

**Доступно:**

- ✅ Создание нового отдела
- ✅ Редактирование названия отдела
- ✅ Редактирование описания отдела (content)
- ✅ Удаление отдела (удаляет все лекции внутри)
- ✅ Просмотр лекций внутри отдела

**Недоступно:**

- ❌ Редактирование отдельных лекций из интерфейса отдела
  - Причина: отдел - это коллекция; правка элементов коллекции должна быть на отдельном уровне

### Визуальное оформление

- **Цветовая схема**: Индиго (indigo) для отличия от уроков (emerald)
- **Иконка**: 📚 + папка (pi-folder)
- **Граница**: `border-indigo-300`
- **Фон**: `bg-indigo-50`

## Компоненты и сервисы

### Сервисы

1. **LectureSectionFormService** (`services/lecture-section-form.service.ts`)
   - Управление формой создания/редактирования отдела
   - CRUD операции для отделов лекций
   - Валидация данных

### Компоненты

1. **LectureSectionFormComponent** (`components/lecture-section-form/`)
   - Форма для создания/редактирования отдела
   - Поля: название (обязательное), описание (необязательное)

2. **Встроенное отображение в основном шаблоне**
   - Отделы лекций отображаются встроенно в `course-lessons-section.html`
   - Избегаем создания суперкомпонента с множеством пропсов
   - Композиция через встраивание в шаблон

## Режимы работы

### Режим просмотра (студент)

- Отображение названия и описания отдела
- Сворачивание/разворачивание списка лекций
- Отметка лекций как просмотренных

### Режим редактирования (преподаватель/админ)

- Включается через `courseLevelEditMode`
- Кнопки "Редактировать отдел" и "Удалить"
- Inline редактирование названия и описания
- Кнопки "Сохранить", "Отмена", "Удалить отдел"

## Состояние UI

Используется `LessonUiStateService` для управления:

- `isLessonCollapsed(sectionId)` - свернут ли отдел
- `isExpanded(sectionId)` - развернут ли список лекций
- `isLectureCollapsed(lectureId)` - свернута ли конкретная лекция

## Сигналы состояния

```typescript
readonly sectionEditMode = signal<string | null>(null); // ID отдела в режиме редактирования
readonly lectureSectionsOnly = computed(() =>
  this.lessons().filter((l) => l.kind === 'lecture')
);
```

## Методы компонента

```typescript
// Создание
openAddLectureSection(): void

// Редактирование
toggleSectionEditMode(sectionId: string): void
isSectionInEditMode(sectionId: string): boolean
cancelSectionEdit(): void
submitLectureSection(): Promise<void>

// Удаление
deleteLectureSection(section: Lesson): Promise<void>
```

## Принципы реализации

1. **Избегаем суперкомпонентов**: Встраиваем отображение прямо в основной шаблон
2. **Композиция через шаблоны**: Используем Angular template syntax для композиции
3. **Разделение ответственности**: Сервисы для логики, компоненты для UI
4. **Явная типизация**: Все инжектируемые сервисы явно типизированы
5. **Сигналы для состояния**: Angular signals для реактивного состояния

## Отличия от уроков

| Аспект     | Уроки (Lessons)             | Отделы лекций (Lecture Sections) |
| ---------- | --------------------------- | -------------------------------- |
| Цвет       | Emerald (зеленый)           | Indigo (синий)                   |
| Иконка     | -                           | 📚 + папка                       |
| kind       | 'lesson'                    | 'lecture'                        |
| Назначение | Учебный материал с лекциями | Группировка лекций по темам      |
